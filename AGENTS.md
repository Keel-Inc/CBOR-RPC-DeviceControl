# AGENTS.md

This file provides guidance to AI coding agents working in this repository.

## Project Overview

Monorepo demonstrating CBOR-RPC communication between a Python host application and STM32F746 Discovery board firmware. The host sends RPC commands over UART (serial-over-TCP via Renode emulator) using CBOR serialization with a 4-byte big-endian length prefix.

## Build & Development Commands

### Device (C firmware for STM32F746)

```bash
# Build (from repo root)
cmake --preset RelWithDebInfo -S Device
cmake --build Device/build/RelWithDebInfo

# Available presets: Debug, Release, RelWithDebInfo, MinSizeRel
```

### Host (Python CLI)

```bash
# Setup
uv venv && source .venv/bin/activate
pip install -e 'Host/.[test,dev]'

# Run tests
cd Host && python -m pytest

# Run a single test
python -m pytest tests/test_cbor_host.py::test_version

# Lint and format
ruff check --fix .
ruff format .
```

### Device Code Formatting

```bash
# Only format non-generated files (not the STM32CubeMX output)
clang-format -i Device/Core/Inc/comm.h Device/Core/Src/comm.c Device/Core/Inc/image.h Device/Core/Src/image.c
```

### Run with Renode Emulator

```bash
renode --console Scripts/device.resc
# USART6 exposed on TCP port 3456
```

## Architecture

**Two main components:**

- **Device/** — Bare-metal C firmware. STM32CubeMX generates HAL setup (`main.c`, peripheral drivers). Custom code lives in `comm.c` (CBOR-RPC message handler) and `image.c` (RGB565 LCD display driver). Uses `tinycbor` submodule (floating-point disabled).

- **Host/** — Python Click CLI (`cbor_host/cli.py`). Subcommands: `test`, `display`, `clear`, `make-header`. Handles image resizing (center-crop to 480x272), RGB565 conversion, CBOR encoding, and serial communication.

**Communication protocol:** CBOR-encoded RPC messages with 4-byte big-endian length prefix over USART6. RPC methods: `test`, `display_image`, `clear_display`, `display_default`.

## Key Conventions

- Device files under `Core/Src/main.c` and `Drivers/` are auto-generated by STM32CubeMX — don't manually edit them.
- The `tinycbor` directory is a git submodule — clone with `--recurse-submodules`.
- CI runs pytest on Python 3.12 (ubuntu-latest). Lint with ruff before committing.
